## 1. `extractTrackerCandidates` 函数完整逻辑

### 函数入口 (第577行)
```go
func extractTrackerCandidates(article model.Article) []trackerCandidate
```

### 完整的多层Pipeline流程

#### **阶段 -1: 《》作品名预提取** (第593-612行)
- 使用 `bookTitleRegex` 正则提取《...》包围的内容(1-20字)
- 用 `forcedEntities` 集合标记,跳过后续启发式判定
- 过滤逻辑:如果是 `isGenericRoleToken` 或 `isGenericEnglishWord`,则跳过
- **目的**: 电影/综艺/书名/歌曲的作品名应强制当 entity,即使未在词典

#### **阶段 -0.5: 「」/『』引用短语预提取** (第614-625行)
- 使用 `quotedPhraseRegex` 提取「...」/『...』内容(1-20字)
- 进入 pool 但**不标记** forced entity,走正常过滤
- **目的**: 知乎标题常用引号引用普通短语(如"肉夹馍""受害者思维"),不强制 entity

#### **阶段 -0.3: 头衔人名预提取** (第627-636行)
- 使用 `honorificRegex` 提取"X+头衔"整段(如"方岱宁院士")
- 正则: `[\p{Han}]{2,4}(院士|教授|总裁|董事长|市长|省长|部长)`
- 标记为 forced entity
- **目的**: GSE 切词把"方岱宁院士"分成两段都识别不出,这里整段强制

#### **阶段 0: 2字合称拆解** (第638-644行)
- 字典: `compoundGeoAbbrevs` (美以/中美/俄乌等)
- 直接在标题中检测合称,展开为两个独立实体
- **例**: "美以袭击"→ 注入 "美国" + "以色列"

#### **阶段 0.5: 动词+地名缩写扫描** (第646-652行)
- 字典: `visitSuffixToEntity` (访华/赴美/访俄等)
- 在标题中检测动宾短语,注入目的地实体
- **例**: "普京启程访华"→ 注入 "中国"(不在标题里但暗示)

#### **阶段 0.55: 人物→关联实体推断** (第654-661行)
- 字典: `personEntityHints` (普京→俄罗斯, 马斯克→特斯拉等)
- 当标题中出现已知人物,自动注入其关联实体
- **例**: "普京启程"→ 注入 "俄罗斯"

#### **阶段 0.6: 复合话题关键词扫描** (第663-670行)
- 字典: `compoundTopicKeywords` (论文打假/校园欺凌/贸易谈判等)
- 直接检测易被分词切散的复合词,整体注入 pool
- **目的**: 防止"论文打假"被切成"论文"+"打假"后短词吸收长词

#### **阶段 1: 词典扫描** (第672-673行)
- 使用 Aho-Corasick 多模匹配(AC自动机)
- 对标题全文不区分大小写扫描 lexicon 别名匹配
- **优先级**: 最高(词典精确命中优先级最高)

#### **阶段 2: 中文分词** (第675-714行)
- 使用 `segmentTitle()` 函数(GSE 中文分词器)
- **分两步处理**:
  1. **词频统计**: 找出现≥2次的词为 `repeatedTokens`
     - 用 `normalizeLexiconAlias`(轻量 trim,不做过滤)统计
     - 原因: weak 词在统计阶段被消灭导致重复检测失效
  2. **进入候选池**: 对每个 token 调 `normalizeTrackerToken`
     - 如果被过滤但在 repeatedTokens 中,用轻量版本替代进入 pool

#### **阶段 3: 兜底处理** (第716-740行)
- **第一步**: 按 `trackerTitleSplitRegex` 拆分成多个 segment
- **第二步**: 对每个 segment 调 `normalizeTrackerToken`
- **第三步**: 在每个 segment 中用 `trackerTokenRegex` 扫描 token
- **兜底**: 如果都没有候选,直接在全标题用正则扫描

### 后处理流程 (第744-785行)

#### **子串去重 Pass 0-2.5** (第774-775行)
调用 `deduplicateCandidateSubstrings()`:
- **Pass 0**: 长 entity(>8字)包含短 entity→删长
- **Pass 1**: keyword 包含 entity→删 keyword
- **Pass 2**: keyword 之间,删长的
- **Pass 2.5**: 复合关键词的子词被吸收

#### **地名形容词过滤** (第777-780行)
调用 `filterAdjunctiveGeoEntities()`:
- 判定地名是否仅以"地名+职业角色"形式出现(如"中国车手")
- 若是,则过滤该地名实体

#### **最终限制** (第782-784行)
- 最多返回 6 个候选(取 8 个后 dedup 保留)

### 候选分类规则 (第745-767行)

对每个 label 进行分类:

```go
kind := "keyword"
if forced || looksLikeEntity(label) {
    kind = "entity"
}
```

**通过条件** (三条任一满足):
1. `forced` - 《》包围的作品名、honorific人名、或人工精选复合词
2. 已重复出现 2 次(`repeated`)- 且不是 stopTokens
3. `shouldKeepTrackerToken(label)` 返回 true

**提取 RelatedTerms** (仅对 entity):
- 调用 `collectTrackerRelatedTerms(ordered, label)`
- 在已有候选中找相关词,最多 4 个

## 2. `looksLikeEntity` 函数 (第1373行)

### 函数目的
判定一个 token 是否看起来像"实体"而非通用词或噪声

### 判定流程 (优先级递减)

1. **词典精确命中** (顶优先级)
   - 检查 `trackerEntityLabelSet` → 直接 return true
   - 强地名 `strongGeoNames` → true

2. **通用词过滤**
   - `isGenericRoleToken()` - stopTokens 中的词 → false
   - `isGenericEnglishWord()` - video/live/special等 → false
   - `isAllGenericEnglishWords()` - "Special Video"这类 → false
   - `isEpisodeCode()` - G1/S22/E3这类集数代号 → false

3. **句子片段拦截**
   - `looksLikeSentence()` - 完整句子、疑问句等 → false
   - `looksLikeDataLikePhrase()` - 赛事统计描述 → false

4. **有大写字母(含非中文字符)** (第1410-1437行)
   - 短纯大写(≤3字 + 全大写) → false (GT/AI/SUV)
   - 长中英混合(汉字>4) → false (描述短串)
   - 1字汉字+英文混合 → false (如"底BOSS")
   - 含动词(hanCount≥2 + 含 `mixedTokenVerbs`) → false (如"发布GPT")
   - 其他(有大写) → true

5. **特殊符号** (中点)
   - 含 "·" → true

6. **后缀匹配** (entitySuffixes)
   - 以"公司/集团/大学/医院/银行/航展"等结尾 → true

7. **前缀匹配** (entityPrefixes)
   - 以"中国/美国/北京"等开头,且剩余长度 1-3 字 → true

8. **兜底**
   - return false

## 3. `shouldKeepTrackerToken` 函数 (第1695行)

### 函数目的
判定一个 token 是否应在最终候选中保留(综合决策)

### 判定流程

```go
func shouldKeepTrackerToken(token string) bool {
    // 1. 词典精确命中 - 最高优先级
    if _, ok := trackerEntityLabelSet[token]; ok {
        return true
    }
    
    // 2. 通用词过滤
    if isGenericRoleToken(token)           { return false } // stopTokens
    if isGenericEnglishWord(token)         { return false } // video/live等
    if isAllGenericEnglishWords(token)     { return false } // "Special Video"
    if isEpisodeCode(token)                { return false } // G1/S22等
    
    // 3. 栏目标签过滤
    if looksLikeChannelTag(token)          { return false } // 【调查】【测评】等
    
    // 4. Entity 判定
    if looksLikeEntity(token)              { return true }
    
    // 5. 强话题名词
    if _, ok := strongTopicNouns[token]; ok { return true }
    
    // 6. 数据/句子片段过滤
    if looksLikeDataLikePhrase(token)      { return false }
    if looksLikeSentence(token)            { return false }
    
    // 7. 兜底:是否看起来像话题短语
    return looksLikeTopicPhrase(token)
}
```

### 保留的 case
- entity (包含所有词典命中、地名、有大写字母的品牌名)
- 强话题名词 (房价/楼市/社保等高频话题词)
- 话题短语 (带 `topicSuffixes` 后缀或是 `strongVerbs` 中的动词)

### 过滤的 case
- stopTokens (回应/发布/知乎等套话)
- 通用英文词 (video/live等)
- 集数编号 (G1/S22)
- 栏目标签 (【调查】等)
- 数据描述 (赛季/比分等)
- 完整句子 (超过 10 汉字、含谓语、疑问句等)

## 6. 正则表达式定义

### `trackerTitleSplitRegex` (第102行)
```go
regexp.MustCompile(`[|｜/:：,，。！？!?()（）\[\]【】<>《》"""''·]+`)
```
**功能**: 标点符号切分标题
**包含**: 管道/竖线/冒号/逗号/句号/问号/括号/方括号/尖括号/书名号/引号/中点

### `bookTitleRegex` (第106行)
```go
regexp.MustCompile(`[《]([^》]{1,20})[》]`)
```
**功能**: 提取《...》包围的作品名
**限制**: 内容 1-20 字,无嵌套
**特殊**: 只有《》包围的内容 force entity
**例**: 《给阿嬷的情书》→ forced entity

### `quotedPhraseRegex` (第110行)
```go
regexp.MustCompile(`[「『]([^」』]{1,20})[」』]`)
```
**功能**: 提取「...」/『...』引用短语
**限制**: 内容 1-20 字
**特殊**: 进入 pool 但不 force entity
**例**: 「肉夹馍」→ 进 pool 走正常过滤

### `honorificRegex` (第116行)
```go
regexp.MustCompile(`[\p{Han}]{2,4}(院士|教授|总裁|董事长|市长|省长|部长)`)
```
**功能**: 提取"X+头衔"组合
**限制**: 人名 2-4 汉字
**特殊**: force entity 处理
**例**: 方岱宁院士 → 整段强制 entity

### `trackerTokenRegex` (第97-101行)
```go
regexp.MustCompile(
    `[#A-Za-z0-9][#A-Za-z0-9+._-]{1,31}` +
    `|[\p{Han}A-Za-z0-9][\p{Han}A-Za-z0-9·.]{1,15}` +
    `|[\p{Han}]{2,12}`
)
```
**功能**: 通用 token 扫描
**三个分支**:
1. `[#A-Za-z0-9][#A-Za-z0-9+._-]{1,31}` - 英文/数字/#起头,最长 32 字
2. `[\p{Han}A-Za-z0-9][\p{Han}A-Za-z0-9·.]{1,15}` - 中英混合,最长 16 字
3. `[\p{Han}]{2,12}` - 纯汉字,2-12 字

### `sentenceFromToRegex` (第1594行)
```go
regexp.MustCompile(`^从.{1,5}到.{1,5}`)
```
**功能**: 标题党"从 X 到 Y"句式
**例**: "从黑马到大爆"

### `episodeCodeRegex` (第1780行)
```go
regexp.MustCompile(`^[A-Za-z]{1,2}\d{1,3}$`)
```
**功能**: 集数/赛事编号(G1/S22/E3)
**限制**: 1-2 个字母 + 1-3 位数字

## 7. 与"event"类别相关的特殊处理

### trackerLexiconEntry 结构 (第83-87行)
```go
type trackerLexiconEntry struct {
    Label    string
    Aliases  []string
    Category string // 元数据:company/person/ip/event/place;空字符串表示未归类
}
```

### event 分类的作用
- **Category 字段**: 前端可据此分类筛选
- **当前代码中**: 在 extractTrackerCandidates 中**没有专门的 event 路径**
- **处理方式**: event 类别通过词典 lexicon entry 标记,前端筛选时使用
- **不影响提取**: 提取逻辑对 company/person/ip/event/place 无区别

### 关键词/实体跟 event 的关系
- `strongVerbs` 中的词(裁员/患癌/确诊/暴跌等)本质是"事件动词"
- `compoundTopicKeywords` 中包含的词(贸易战/校园欺凌等)通常标记为 event
- `topicSuffixes` 中的词(事件/政策/比赛/事故等)直接包含"事件"语义

## 10. `dataLikePatterns` 正则列表

### 用途
识别"伪体育/数据描述"片段,是统计文案而非事件或实体,应过滤掉

### 现有模式 (第1482-1523行, 共23个正则)

#### **赛季/轮次描述** (4个)
```
`^\d{2,4}\s*赛季.+第\s*\d+\s*轮`           - "2526赛季英超第37轮"
`^\d{2,4}\s*赛季`                         - "2526赛季"开头
`赛季.{2,8}第\s*\d+\s*轮`                 - "赛季...第N轮"
`第\s*\d+\s*轮`                          - "第37轮"
```

#### **赛事类型描述** (2个)
```
`^赛季.+(季后赛|联赛|总决赛|常规赛)`      - "赛季...季后赛"
`\d+\s*[-:：]\s*\d+\s*战胜`               - "1-0 战胜"比分赛事
```

#### **战队/比分** (3个)
```
`^战胜.{2,6}$`                           - "战胜伯恩利"
`.{2,8}\s+\d+\s*[-:：]\s*\d+\s+.{2,8}`  - "马刺 122-115 雷霆"
```

#### **运动员数据** (1个)
```
`.{2,8}\s*\d+\s*分\s*\d+\s*(篮板|助攻|抢断|盖帽|出场|首发)` - "文班亚马 41 分 24 篮板"
```

#### **系列赛编号** (1个)
```
`^.{1,4}决\s*G\d+$`                     - "西决G1" / "东决G3"
```

#### **票房/冠军** (2个)
```
`票房破\s*\d+\s*亿`                      - "票房破5亿"
`累计.{0,4}破\s*\d+`                     - "累计破10亿"
```

#### **排名/标题** (3个)
```
`^第\s*\d+\s*冠$`                        - "第5冠"
`机车夺.*第\s*\d+\s*冠`                  - "机车夺...第N冠"
`\d{4}年\s*\d+月\s*\d+号`                - 完整日期
```

#### **赛事名编码** (1个)
```
`^[A-Z]{2,5}\d{4}$`                    - "LPL2026" "S22"
```

#### **公示句式** (1个)
```
`公布.{2,10}(名单|阵容|排名|榜单)` - "公布26人名单"
```

#### **数量描述** (2个)
```
`含\s*\d+\s*位.{2,10}`                 - "含5位副总理"
`\d+\s*位(副?总理|部长|院士|教授|代表|嘉宾|官员|博士)` - "8位部长"
`^位(副?总理|部长|院士|教授|代表|嘉宾|官员|博士)$` - "位部长"
```

#### **人物数据** (3个)
```
`^\d{1,3}岁.{1,8}领衔$`                - "41岁C罗领衔"
`^\d{1,3}岁.{1,6}(首发|入选|进球|出场)` - "33岁梅西首发"
`^岁.{1,8}领衔$`                       - "岁...领衔"(头部数字被切掉)
```

### looksLikeDataLikePhrase 函数 (第1525行)
```go
func looksLikeDataLikePhrase(token string) bool {
    for _, re := range dataLikePatterns {
        if re.MatchString(token) {
            return true
        }
    }
    return false
}
```
